1. Назначение и архитектура

1.1 Что такое ScaleKiosk

ScaleKiosk — полноэкранный модальный интерфейс для операторов промышленных весов, оптимизированный для тачскринов. Используется в двух режимах:

РежимМодульHTML-шаблонПеременная
purchase (приёмка)AGRO Fieldagro_field.htmlfieldKiosk
sale (отгрузка)AGRO Salesagro_sales.htmlsalesKiosk

1.2 Принцип «один класс — два режима»

Класс написан по шаблону IIFE (Immediately Invoked Function Expression), экспортируется в window.ScaleKiosk. Различия между режимами контролируются конфигурационными флагами:

├──────────────────────────────────────┤
│  Config flags:                       │
│    mode: 'purchase' | 'sale'         │

│    showEmulator: true | false        │
│    products: [{key,name,svgPath}]    │

4. Публичный API

4.1 .open(lineId)

Открывает полноэкранный модал и привязывает его к строке документа.

kiosk.open('line_42');

Действия:

  1. Сохраняет lineId в this._targetLineId
  2. Сбрасывает состояние: _selectedProduct, _selectedVariety, _manualWeight, _paused
  3. Устанавливает UI в начальное состояние (вес 0.000, статус idle, capture disabled)
  4. Если showPassport — скрывает паспорт
  5. Если loadDriverConfig — загружает конфиг драйвера
  6. Строит грид продуктов
  7. Показывает модал (display: flex)
  8. Запускает polling весов (1с)
  9. Запускает авто-эмуляцию (20с)

4.2 .close()

Закрывает модал, останавливает polling и эмуляцию, сбрасывает состояние.

4.3 .destroy()

Вызывает .close(), удаляет DOM-элемент контейнера из document.body.

13. Авто-эмуляция

.open() → _startAutoEmulation() → setInterval(_runEmulationCycle, 20000)

Шаги одного цикла:

  1. Если _paused → return
  2. Показать анимацию сканирования
  3. POST /api/agro-scale/simulate (random weight)
  4. Ожидание стабильности (300мс × 20 попыток)
  5. Случайный продукт
  6. Confidence 85–99%
  7. Update: viewport, AI result, highlight product, show passport
  8. Разблокировать Capture

│           │                  │                   │

15. Процесс захвата (Capture)

15.1 Последовательность

1. captureBtn.disabled = true

4. POST /api/agro-scale/capture {scale_id}
5. reading: {gross_kg, tare_kg, net_kg}
6. data = {lineId, gross, tare, net, productKey, productName, passport}
7. onCapture(data)
8. close()
9. toast: «Вес снят: X.XX кг — Продукт»

16. Конфигурация драйвера

GET /api/agro-admin/module-config
  → filter config_group === 'scale_driver'
    → parse JSON → find active !== false
      → title: '⚖️ Весы — {brand}'
      → driverInfo: 'Driver: {brand} | Max: {max_capacity} kg'

Fallback: Driver: Emulator | Max: 600 kg

24. Диаграмма жизненного цикла

[Конструктор]
    ├── Читает config
    ├── Строит product maps
    ├── _renderModal() → DOM добавлен в body (display:none)
    └── _bindEvents() → delegation на container

[.open(lineId)]
    ├── Reset state + UI
    ├── _hidePassport()
    ├── _loadDriverConfig() → GET /api/agro-admin/module-config
    ├── _buildProductGrid()
    ├── container.style.display = 'flex'
    ├── _startPolling() → setInterval(1000ms)
    │     └── _pollScale() → GET /api/agro-scale/read
    └── _startAutoEmulation() → setInterval(20000ms)
          └── _runEmulationCycle()

[Работа оператора]
    ├── Click product → _selectProduct() → _showPassport()
    ├── Fill passport → _stepField(), _setRating(), _selectOpt()
    ├── Numpad → _numpadPress() + _numpadApply()
    ├── Zero → _scaleZero(), Tare → _scaleTare()
    └── Pause → _togglePause()

[Capture]
    ├── captureBtn disabled, _stopAutoEmulation()
    ├── POST /api/agro-scale/capture
    ├── data = {lineId, gross, tare, net, productKey, passport}
    ├── onCapture(data)
    ├── close()
    └── toast: «Вес снят: X.XX кг — Продукт»

[.close()]
    ├── display:none
    ├── _stopPolling(), _stopAutoEmulation()
    ├── Reset state
    └── _hidePassport()

[.destroy()]
    ├── close()
    └── Remove DOM, _container = null
