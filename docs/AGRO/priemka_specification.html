2. Журнал документов (стиль 1С)

Журнал построен по принципу master-detail — верхняя таблица показывает шапки документов, нижняя панель — содержимое выбранного документа. Этот паттерн аналогичен журналу документов в 1С:Предприятие.

2.1. Вкладки статусов

Горизонтальная полоса вкладок .status-tabs позволяет фильтровать документы по статусу. Каждая вкладка показывает счётчик.

Вкладкаfilter-valueСчётчик IDОписание
ВсеallcountAllПоказать все документы
ЧерновикDRAFTcountDraftНе подтверждённые документы
Подтв.CONFIRMEDcountConfirmedПодтверждённые документы
ЗакрытCLOSEDcountClosedЗакрытые/архивные документы

CSS: активная вкладка — .status-tab.active (белый текст, яркий нижний бордер). Счётчик — .tab-count (маленький badge).

JS: клик вызывает filterPurchases(status), который обновляет UI и перезагружает данные.

2.2. Поиск

Поле <input class="search-input"> под вкладками. Фильтрует по:

  • Номеру документа
  • Имени поставщика / Furnizor
  • Дате

Фильтрация — клиентская, мгновенная (событие oninput).

  • Badge-стили: .badge-draft (серый), .badge-confirmed (зелёный), .badge-closed (голубой)

2.4. Детальная панель

Блок .journal-detail (#purchaseDetailPanel) под мастер-таблицей. По умолчанию сообщение «Выберите документ из списка».

Содержимое при выборе документа:

  • Номер документа (крупно) + дата + поставщик + склад + статус badge
  • Таблица позиций: Продукт | Кол-во кг | Цена/кг | Сумма
  • Итоговая сумма
  • Кнопка «Открыть» (для перехода к редактированию)

Layout: .journal-master-detail — вертикальный flex с gap:0. Мастер — верхняя половина (~50%), детали — нижняя.

2.5. Встроенная инструкция

Сворачиваемая панель .instructions-panel с заголовком «📖 Инструкция / Instructiuni de utilizare». Содержит 6 пунктов (нумерованный список ol):

  1. В верхней таблице — список документов. Кликните для просмотра.
  2. Используйте вкладки статусов для фильтрации.
  3. Поле поиска фильтрует по номеру, поставщику и дате.
  4. В нижней панели — строки выбранного документа.
  5. Для редактирования нажмите «Открыть» в нижней панели.
  6. Кнопка + Новая закупка создаёт новый документ.

4.2. Левая панель: Продукция (тач-кнопки)

Грид .sk-product-grid с квадратными кнопками продуктов. Каждая кнопка:

  • Иконка продукта (img)
  • Название (RU / RO)
  • Клик → выделяет продукт, показывает паспорт

Грид: grid-template-columns: repeat(auto-fill, minmax(80px, 1fr))

Интеграция: при захвате веса выбранный продукт автоматически заполняет .line-item (select) в строке документа.

4.3. Центральная панель: Весы + Numpad

Дисплей веса:

  • .sk-weight-gross — огромный шрифт (в field 56px, в практике ~40-56px), зелёный, font-family: 'Courier New', monospace
  • Единица: .sk-weight-unit — "kg"
  • Субэлементы: .sk-weight-sub — Тара и Нетто в двух боксах

Кнопки управления:

  • Zero — modalScaleZero() → POST /api/agro-scale/zero
  • Tare — modalScaleTare() → POST /api/agro-scale/tare

Numpad: .sk-numpad — ручной ввод веса.

  • 3×4 грид кнопок: 7-8-9, 4-5-6, 1-2-3, 0-.-⌫
  • Текстовое поле #numpadInput (readonly — только через кнопки)
  • Кнопка «Применить / Aplica» — numpadApply()

Главная кнопка захвата: .sk-capture-btn — «⚖️ Снять вес / Cântărește»

  • Красная, крупная (min-height:60px, font-size:18px)
  • Заблокирована (disabled) пока статус не «stable»
  • Вызывает modalScaleCapture()

4.4. Правая панель: AI Camera

Блок .sk-camera — камера с AI-распознаванием продукции.

  • Заголовок: «📷 AI Camera» + badge «AI»
  • Viewport #cameraViewport: показывает анимацию сканирования, затем pictogram + confidence bar
  • Результат: #aiResult — «Распознано: [Продукт]», «Точность: 95.2% | Вес: 123.45 кг»

Авто-эмуляция: каждые 20 секунд:

  1. Симуляция случайного веса (50-500 кг)
  2. Показ анимации сканирования
  3. Выбор случайного продукта
  4. Показ pictogram + confidence в viewport
  5. Выделение продукта в гриде + показ паспорта

4.6. Рабочий процесс взвешивания

  1. Оператор нажимает ⚖ в строке документа
  2. Открывается полноэкранный модал
  3. Начинается автоматический polling весов (setInterval(pollModalScale, 1000))
  4. Запускается авто-эмуляция (каждые 20с — новый вес + AI-распознавание)
  5. Вес отображается в реальном времени:
    • Серый индикатор #666 → Idle
    • Жёлтый #ffc107 → Settling (стабилизация)
    • Зелёный #53d769 → Stable (кнопка Capture разблокируется)
  6. Оператор может:
    • Выбрать продукт из грида (или AI выберет автоматически)
    • Заполнить паспорт продукции
    • Ввести вес вручную через numpad
    • Нажать Zero / Tare
  7. Нажатие Capture:
    • POST /api/agro-scale/capture
    • Gross/Tare заполняются в строку документа
    • Если выбран продукт — заполняется .line-item (select)
    • Данные паспорта сохраняются в data-passport
    • recalcPurchaseLine(lineId) пересчитывает нетто и сумму
    • Модал закрывается
    • Toast: «Вес снят: X.XX кг — Продукт»

5. Страница весов (автономная)

Страница page-scale — автономный интерфейс для работы с весами вне контекста документа.

Блоки:

  1. Live вес — индикатор статуса (кружок), текст статуса, gross (72px шрифт), тара/нетто боксы
  2. Кнопки: Zero / Tare / Capture
  3. Эмулятор: поле ввода веса, кнопки «Поставить», «Случайный», «Снять»
  4. Зафиксированный вес: (скрыт до capture) — карточка с gross/tare/net + кнопки «Применить к строке» / «Сбросить»
  5. Конфигурация: max capacity, resolution, settle time, mode

Polling: setInterval(pollScale, 1000) — идёт всё время, пока страница активна.

6. Страница сканирования (Scan / Приёмка)

Страница page-scan — сканирование штрихкодов ящиков.

  • Вьюфайндер камеры — .camera-viewfinder
  • Поле штрихкода: ручной ввод + кнопка «Найти»
  • Панель ящика:
    • Тип упаковки (select: Стандартный ящик, Паллет, Картон, Мешок)
    • Брутто кг (input number)
    • Тара кг (display/input)
    • Нетто кг (auto-calculated)
  • Кнопка «Сохранить» → привязывает вес к ящику

7. Штрихкоды

Страница page-barcodes — генерация и печать штрихкодов для ящиков.

  • Ввод количества + кнопка «Генерировать»
  • Таблица: чекбокс | Штрихкод (mono) | Статус | Дата
  • Кнопки: «Выбрать все» / «Печатать выбранные»
  • Print: скрытый контейнер #barcodePrintContainer — формат для термо-принтера

8. Заявки на закупку

Страница page-requests — управление заявками поставщикам.

  • Фильтр: select с статусами (Все / Черновик / Утверждена / Выполнена / Отменена)
  • Таблица: №, Дата, Поставщик, Склад, Профиль, Ожид. кг, Строк, Статус
  • Форма создания: hidden (открывается кнопкой «+ Новая заявка»)
  • Позиции заявки: Продукт + Кол-во кг + Сорт + Примечание
  • Действия: Сохранить / Утвердить
