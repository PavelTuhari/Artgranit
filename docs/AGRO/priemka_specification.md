## 1. Архитектура и навигация

### 1.1 Боковое меню (Sidebar)

Меню разбито на секции с разделителями. Все пункты — `<a class="nav-link" data-page="...">`. Активная страница — CSS `.active`.

| Секция | Пункт меню | data-page | Описание |
|--------|-----------|-----------|----------|
| **Операции / Operatii** | Scan / Приёмка | `scan` | Камера + ручной ввод штрихкода |
| | Закупка / Achizitie | `purchase` | Форма создания документа закупки |
| | Штрихкоды / Coduri | `barcodes` | Генерация и печать штрихкодов |
| | Список / Lista | `purchases-list` | Журнал документов (стиль 1С) |
| **Весы / Cântar** | Весы / Cântar | `scale` | Автономная страница весов |
| **Заявки и приёмка** | Заявки / Cereri | `requests` | Заявки поставщикам |
| | Приёмка партий / Receptie | `inspections` | Контроль качества партий |

### 1.2 Страницы модуля

Каждая страница — `div.page`. Переключение через `navigateTo(pageName)`.

| Page ID | Название | Контент |
|---------|---------|---------|
| `page-scan` | Scan / Приёмка | Камера, штрихкод, панель ящика (тип, брутто, тара, нетто) |
| `page-purchase` | Закупка | Шапка + табличная часть + итого + Draft/Confirm |
| `page-barcodes` | Штрихкоды | Генерация N штрихкодов, таблица, печать |
| `page-purchases-list` | **Журнал** | Master-detail journal (→ §2) |
| `page-scale` | Весы | Live вес (72px), Zero/Tare/Capture, эмулятор |
| `page-requests` | Заявки | Фильтр статусов, таблица, форма создания |
| `page-inspections` | Приёмка партий | Партия, чек-лист (4 группы), D-коды, фото, история |

## 2. Журнал документов (стиль 1С)

Журнал построен по принципу **master-detail** — верхняя таблица = шапки документов, нижняя панель = содержимое выбранного. Аналог журнала в 1С:Предприятие.

### 2.1 Вкладки статусов

Горизонтальная полоса `.status-tabs`. Каждая вкладка показывает счётчик.

| Вкладка | filter-value | Counter ID | Описание |
|---------|-------------|------------|----------|
| **Все** | `all` | `countAll` | Все документы |
| **Черновик** | `DRAFT` | `countDraft` | Не подтверждённые |
| **Подтв.** | `CONFIRMED` | `countConfirmed` | Подтверждённые |
| **Закрыт** | `CLOSED` | `countClosed` | Архивные |

**CSS:** `.status-tab.active` — белый текст, яркий нижний бордер. `.tab-count` — маленький badge.

**JS:** клик → `filterPurchases(status)`.

### 2.2 Поиск

`<input class="search-input">` под вкладками. Клиентская фильтрация (`oninput`) по:
- Номеру документа
- Имени поставщика
- Дате

### 2.3 Мастер-таблица

Блок `.journal-master`:

| Колонка | Ширина | Align | Описание |
|---------|--------|-------|----------|
| **№** | 80px | left | Номер документа |
| **Дата** | 110px | left | Дата |
| **Поставщик** | auto | left | Имя контрагента |
| **Склад** | 90px | left | Склад приёмки |
| **Статус** | 100px | left | Badge (DRAFT/CONFIRMED/CLOSED) |
| **Кг** | 90px | right | Итого нетто |
| **Сумма** | 100px | right | Итого в валюте |

**Поведение:**
- Клик по строке → загружает подробности в детальную панель
- `.journal-master tr:hover` — подсветка
- Badge-стили: `.badge-draft` (серый), `.badge-confirmed` (зелёный), `.badge-closed` (голубой)

### 2.4 Детальная панель

`.journal-detail` (`#purchaseDetailPanel`) под мастер-таблицей.

**Пустое состояние:** «Выберите документ из списка»

**Содержимое при выборе:**
- Номер документа (крупно) + дата + поставщик + склад + статус badge
- Таблица позиций: Продукт | Кол-во кг | Цена/кг | Сумма
- Итоговая сумма
- Кнопка «Открыть»

**Layout:** `.journal-master-detail` — вертикальный flex. Мастер ~50%, детали ~50%.

### 2.5 Встроенная инструкция

Сворачиваемая панель `.instructions-panel`:

1. В верхней таблице — **список документов**. Кликните для просмотра.
2. Используйте **вкладки статусов** для фильтрации.
3. Поле **поиска** фильтрует по номеру, поставщику и дате.
4. В **нижней панели** — строки выбранного документа.
5. Для **редактирования** нажмите «Открыть» в нижней панели.
6. Кнопка `+ Новая закупка` создаёт новый документ.

### 3.2 Табличная часть (позиции)

| Колонка | Тип | CSS-класс | Описание |
|---------|-----|-----------|----------|
| Продукция | `<select>` | `.line-item` | Справочник items |

| Ящики | `<input number>` | `.line-crates` | Количество |
| Брутто | `<input number>` | `.line-gross` | Заполняется весами или вручную |
| Тара | `<input number>` | `.line-tare` | Заполняется весами или вручную |
| Нетто | display only | — | gross − tare |
| Цена/кг | `<input number>` | `.line-price` | Цена за кг |
| **⚖ Весы** | `<button>` | `.btn-weigh` | **Открывает Scale Kiosk Modal** |

### 3.3 Действия

- **Черновик / Draft** (`btn-ghost`) — `savePurchaseDraft()`
- **Подтвердить / Confirma** (`btn-primary`) — `confirmPurchase()`

## 4. Точскрин-режим весов (Scale Kiosk Modal)

> **КРИТИЧЕСКИ ВАЖНО** — ключевой интерфейс для операторов весов.
>
> **Рефакторинг (28.03.2026):** Scale Kiosk вынесен в универсальный класс `ScaleKiosk`
> (`/static/agro/scale-kiosk.js` + `/static/agro/scale-kiosk.css`).
> Оба модуля (приёмка и продажа) создают экземпляр класса с нужной конфигурацией:
> - Приёмка: `new ScaleKiosk({mode:'purchase', showPassport:true, showEmulator:false})`
> - Продажа: `new ScaleKiosk({mode:'sale', showPassport:true, showEmulator:true})`

CSS: `.sk-body { display:flex; gap:32px }` — три колонки: `.sk-products` + `.sk-scale` + `.sk-camera`.

### 4.2 Левая панель: Продукция

`.sk-product-grid` — грид тач-кнопок с иконками/названиями продуктов.
- Grid: `repeat(auto-fill, minmax(80px, 1fr))`
- Клик → выделяет продукт, показывает паспорт
- При захвате выбранный продукт → `.line-item` (select) в строке документа

### 4.3 Центральная панель: Весы + Numpad

**Дисплей веса:**
- `.sk-weight-gross` — 56px, зелёный, monospace
- `.sk-weight-unit` — "kg"
- `.sk-weight-sub` — Тара и Нетто боксы

**Управление:**
- `Zero` → POST `/api/agro-scale/zero`
- `Tare` → POST `/api/agro-scale/tare`

**Numpad:** `.sk-numpad`
- 3×4 грид: 7-8-9, 4-5-6, 1-2-3, 0-.-⌫
- `#numpadInput` (readonly)
- «Применить / Aplica» — `numpadApply()`

**Capture:** `.sk-capture-btn`
- Красная, крупная (min-height:60px, font-size:18px)
- Disabled пока статус ≠ stable
- `modalScaleCapture()` → POST `/api/agro-scale/capture`

### 4.4 Правая панель: AI Camera

`.sk-camera`:
- Viewport: анимация сканирования → pictogram + confidence bar
- Результат: «Распознано: [Продукт]», «Точность: XX% | Вес: XX кг»

**Авто-эмуляция** (каждые 20с):
1. Симуляция случайного веса (50-500 кг)
2. Анимация сканирования
3. Случайный продукт
4. Pictogram + confidence в viewport
5. Выделение продукта в гриде + показ паспорта

### 4.5 Паспорт продукции

`.sk-passport` — под гридом продуктов. Тач-оптимизированные поля:

| Поле | Виджет | ID |
|------|--------|-----|
| Сорт / Soi | Грид кнопок | `passportVarietyGrid` |
| Калибр, мм | Stepper (−/+) | `ppCalibr` |
| Brix, ° | Stepper (−/+) | `ppBrix` |
| Окраска, % | Range slider | `ppColorSlider` |
| t °C | Stepper (−/+) | `ppTemp` |
| Свежесть | Rating 1-5 | `ppFreshness` |
| Срок хран. | Readonly | `ppShelfLife` |
| Дефекты | Btn-group (Нет/Мин./Серьёз.) | `ppDefects` |
| % дефектных | Stepper | `ppDefectPct` |
| Упаковка | Btn-group (OK/Поврежд./Нет) | `ppPackaging` |
| Маркировка | Btn-group (OK/Частичн./Нет) | `ppLabel` |
| Заметки | Textarea | `ppNotes` |

При захвате — данные сохраняются в `data-passport` атрибут строки.

### 4.6 Рабочий процесс взвешивания

1. Оператор нажимает **⚖** в строке документа
2. Открывается полноэкранный модал
3. Polling весов: `setInterval(pollModalScale, 1000)`
4. Авто-эмуляция: 20с цикл (вес + AI)
5. Индикация статуса:
   - 🔘 Серый `#666` → Idle
   - 🟡 Жёлтый `#ffc107` → Settling
   - 🟢 Зелёный `#53d769` → Stable (Capture разблокирован)
6. Оператор может: выбрать продукт / заполнить паспорт / ввести вес numpad / Zero / Tare
7. **Capture:**
   - POST `/api/agro-scale/capture`
   - Gross/Tare → строка документа
   - Продукт → `.line-item`
   - Паспорт → `data-passport`
   - `recalcPurchaseLine(lineId)`
   - Модал закрывается
   - Toast: «Вес снят: X.XX кг — Продукт»

### 4.7 API весов

| Method | Endpoint | Description | Body |
|--------|---------|-------------|------|
| GET | `/api/agro-scale/read` | Текущее состояние | `?scale_id=default` |
| POST | `/api/agro-scale/capture` | Захватить вес | `{ scale_id }` |
| POST | `/api/agro-scale/zero` | Обнуление | `{ scale_id }` |
| POST | `/api/agro-scale/tare` | Установить тару | `{ scale_id }` |
| POST | `/api/agro-scale/simulate` | Эмулятор | `{ scale_id, weight_kg?, random?, min_kg?, max_kg? }` |

**Ответ `/read`:**
```json
{
  "success": true,
  "data": {
    "gross_kg": 125.500,
    "tare_kg": 0.000,
    "net_kg": 125.500,
    "status": "stable | settling | idle",
    "stable": true,
    "timestamp": "2026-03-28T14:30:00"
  }
}
```

## 5. Страница весов (автономная)

`page-scale` — работа с весами вне документа:

1. **Live вес** — индикатор (кружок), статус, gross (72px), тара/нетто боксы
2. **Кнопки:** Zero / Tare / Capture
3. **Эмулятор:** поле веса + Поставить / Случайный / Снять
4. **Зафиксированный вес** (скрыт до capture) — gross/tare/net + Применить / Сбросить
5. **Конфигурация:** max capacity, resolution, settle time, mode

Polling: `setInterval(pollScale, 1000)`.

## 12. GAP-анализ: Отгрузка vs Приёмка

| Функция | Приёмка (Field) | Отгрузка (Sales) | Статус |
|---------|----------------|------------------|--------|
| **Боковое меню** | 3 секции (7 пунктов) | 3 секции (6 пунктов) | ✅ OK |
| **Журнал (master-detail)** | Tabs + search + 7 col + detail + instructions | Tabs + search + 6 col + detail | ⚠️ Нет инструкции, нет колонки Кг |
| **Форма документа** | 3 поля + 9-col grid + Draft/Confirm | 4 поля + 8-col grid + Draft/Confirm/Allocate | ✅ OK |
| **⚖ → модал** | Scale Kiosk Modal | Scale Kiosk Modal (добавлено) | ✅ OK |
| **Scale: 3-колонки** | Products + Scale + AI Camera | Только Scale (1 кол.) | ⚠️ Упрощённый |
| **Auto-emulation** | 20с цикл (вес + AI) | Нет | ⚠️ Можно добавить |
| **Паспорт продукции** | 12 полей | Нет (не нужно) | N/A |
| **Pause button** | ⏸ в хедере модала | Нет | ❌ Отсутствует |
| **Driver info** | в хедере модала | Нет | ❌ Отсутствует |
| **Инструкция в журнале** | Сворачиваемая, 6 пунктов | Нет | ❌ Отсутствует |
| **Колонка «Кг» в журнале** | Есть | Нет | ❌ Отсутствует |
| **Автономные весы** | Полная страница | Упрощённая | ⚠️ |
| **Акты (tickets)** | Нет | Есть | ✅ Sales-specific |
