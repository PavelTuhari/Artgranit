Textul specificațiilor tehnice

1. Introducere

Modulul AGRO gestionează ciclul complet al operațiunilor cu produse agricole:

Recepție câmp → Depozitare frigorifică → Control calitate / HACCP → Procesare → Vânzări / Export

Caracteristici cheie

  • Prefix Oracle: AGRO_
  • 36 tabele normalizate + 5 vizualizări + 36 triggere auto-ID + 3 triggere de audit
  • 5 interfețe UI cu roluri distincte: Admin, Recepție, Depozit, Calitate, Vânzări
  • Evenimente în timp real via Socket.IO (alerte temperatură, notificări blocuri)
  • Alocare loturi FIFO automată la confirmarea documentelor de vânzare
  • 9 tipuri de documente tipărite (acte, tichete, facturi, declarații vamale)

Domenii funcționale

  • Recepție câmp — achiziții de la producători, cântărire lăzi, generare coduri de bare EAN/QR
  • Depozit frigorific — stocuri pe celule, mișcări de intrare/ieșire, monitorizare temperatură și umiditate
  • Calitate și HACCP — liste de verificare GMP, controale fizico-chimice, blocări loturi la neconformitate, planuri HACCP cu puncte critice de control
  • Vânzări și export — documente de vânzare, alocare loturi, declarații vamale pentru export

Pagina AGRO Admin — secțiuni

  • Date de referință — furnizori, clienți, depozite, celule, articole, tipuri ambalaj, vehicule, valute, cursuri valutare, parametri formulă, configurare modul
  • Rapoarte — achiziții, vânzări, bilanț de masă, stoc curent, expirări
  • Export — orice raport în format XLSX sau CSV

Pagina AGRO Recepție (Field) — secțiuni

  • Coduri de bare — generare lot, tipărire etichete EAN-128/QR
  • Lăzi (Crates) — scanare cod de bare, înregistrare greutate brut/tară/net
  • Documente achiziție — creare, confirmare, anulare; sincronizare offline pentru tabletă în câmp

Pagina AGRO Depozit (Warehouse) — secțiuni

  • Stoc curent — solduri pe lot / depozit / celulă cu dată expirare și status lot
  • Mișcări — intrare, ieșire, transfer intern între celule
  • Temperaturi și umiditate — citiri senzori, grafice istorice, alerte la depășire prag
  • Sarcini depozit — gestionarea task-urilor operatorilor

Pagina AGRO Calitate / HACCP (QA) — secțiuni

  • Liste de verificare — șabloane GMP cu puncte de control configurabile
  • Controale calitate — execuție verificări pe loturi cu valori măsurate
  • Blocări loturi — blocare automată la neconformitate critică; deblocare cu justificare
  • HACCP — planuri HACCP, puncte critice de control (CCP), înregistrări și devieri

Pagina AGRO Vânzări (Sales) — secțiuni

  • Documente vânzare — creare, confirmare cu alocare FIFO automată, tipărire factură
  • Stoc disponibil — vizualizare loturi disponibile pentru vânzare
  • Export și declarații vamale — creare și gestionare declarații de export

7. Configurare locală

Pas 1 — Configurare Oracle Wallet în .env

WALLET_DIR=/path/to/oracle_wallet
DB_USER=your_user
DB_PASSWORD=your_password
DB_DSN=your_dsn
PORT=3003

Pas 2 — Deploiere DDL (creare tabele Oracle)

python deploy_oracle_objects.py

Se execută în ordine: sql/35_agro_tables.sql, sql/36_agro_views.sql, sql/37_agro_triggers.sql, sql/38_agro_demo_data.sql.

Pas 3 — Lansarea aplicației

./run_local.sh
# sau direct:
python app.py

Aplicația pornește pe portul configurat în .env (implicit PORT=3003).

Pas 4 — Navigare în browser

  • MDI Shell: http://localhost:3003/UNA.md/orasldev/agro
  • Admin: http://localhost:3003/UNA.md/orasldev/agro-admin
  • Recepție: http://localhost:3003/UNA.md/orasldev/agro-field
  • Depozit: http://localhost:3003/UNA.md/orasldev/agro-warehouse
  • Calitate/HACCP: http://localhost:3003/UNA.md/orasldev/agro-qa
  • Vânzări: http://localhost:3003/UNA.md/orasldev/agro-sales

Notă privind Oracle Wallet local

Oracle Wallet nu face parte din codul sursă al aplicației. WALLET_DIR din .env trebuie să fie un path absolut către wallet-ul local. Nu se face commit cu wallet-ul în repozitoriu.

8. Deploiere pe serverul remote

Pas 1 — Transfer cod

./deploy_to_remote.sh

Transferă codul aplicației la /home/ubuntu/artgranit pe serverul remote. Nu execută DDL Oracle.

Pas 2 — Oracle Wallet pe remote

Wallet-ul Oracle rămâne la locația sa permanentă pe server, în afara directorului de deploy:

/home/ubuntu/oracle_wallets/wallet_HXPAVUNKCLU9HE7Q

În .env de pe server, WALLET_DIR trebuie să indice acest path absolut.

Pas 3 — Deploiere modificări schemă Oracle (dacă e cazul)

Dacă release-ul include tabele noi sau modificări DDL, se execută separat:

DEPLOY_ORACLE_ON_REMOTE=1 python deploy_oracle_objects.py

Pas 4 — Verificare URL producție

  • MDI Shell: http://92.5.3.187:8000/UNA.md/orasldev/agro

9. Verificare post-deploiere (Checklist)

Obiecte Oracle

  • Obiectele Oracle cu prefixul AGRO_ sunt vizibile în USER_OBJECTS
  • Toate cele 36 tabele, 5 vizualizări, 36 triggere auto-ID și 3 triggere audit există

Interfețe UI

  • Toate cele 5 pagini UI se încarcă fără erori (agro-admin, agro-field, agro-warehouse, agro-qa, agro-sales)
  • MDI Shell /agro afișează tab-urile tuturor modulelor

Funcționalitate Admin

  • CRUD funcțional pentru toate cele 11 tabele de referință (furnizori, clienți, depozite, celule, articole, tipuri ambalaj, vehicule, valute, cursuri, parametri, configurare)
  • Rapoartele generează date corecte pentru toate cele 5 tipuri (achiziții, vânzări, bilanț masă, stoc, expirări)
  • Export funcțional în format XLSX și CSV

Flux Recepție

  • Generarea codurilor de bare produce valori unice
  • Scanarea unui cod de bare identifică corect lada
  • Cântărirea lăzii salvează greutatea și calculează netul
  • Crearea și confirmarea documentului de achiziție creează loturile și mișcările de stoc corespunzătoare

Flux Depozit

  • Soldurile de stoc se actualizează corect după mișcări
  • Citirile de temperatură se salvează și apar în grafice
  • Depășirea pragului de temperatură generează alertă automată
  • Confirmarea alertei (ack) funcționează

Flux Calitate / HACCP

  • Crearea și execuția unui control de calitate pe lot funcționează
  • O neconformitate critică blochează automat lotul
  • Deblocarea lotului necesită justificare și funcționează corect
  • Înregistrările CCP cu deviere apar în lista de devieri

Flux Vânzări

  • Confirmarea documentului de vânzare execută alocarea FIFO corect
  • Stocul se reduce după confirmare
  • Loturile blocate nu sunt incluse în stocul disponibil pentru vânzare
  • Declarația de export se creează și se poate actualiza

Documente tipărite

  • Toate cele 9 tipuri de documente se afișează corect în format A4
  • Imprimarea din browser funcționează fără elemente de navigare
