🧾 ServOuts26 — CRM/SaaS servicii de contabilitate

Модуль платформы Artgranit · схема UNITEST (Oracle 11g, тот же сервер, что OfficePlus) · трёхъязычный интерфейс RU/RO/EN · бизнес-логика в пакете YServOuts_BP

1. Назначение

CRM-инструменты для бухгалтерской фирмы поверх существующей ERP-схемы UNITEST: справочник услуг и позиций, клиенты/организации, формирование цен на услуги (прайс-листы), конфигурируемый импорт фидов и журнал операций. Фаза 2 (перспектива, ТЗ §10): документы приходов/продаж TMDB_*, счета к оплате с отправкой на email, SMS о задолженностях.

4. Пакет YServOuts_BP

Конфигурация — публичные переменные g_* (таблица/колонки источника, секвенция, CODPRICE, константы UM/GR1/TIP/CACCESS/CODTVA, даты, лимиты длин). Чтение/запись по имени: get_conf(name) / set_conf(name, value); идентификаторы валидируются (assert_idents) перед динамическим SQL.

Процедуры: prepare_input → validate_input → assign_keys → import_univers → import_groups → import_dates → import_prices (или всё сразу — import_all), rollback_pricelist(cp) (цены → периоды → именованные группы → пустая шапка; нативные данные не трогаются), merge_groups(src, dst, cp), archive_univers(cod, val), fix_denumirea_confusables (замена кириллических «обманок» С→C при ≤ g_confus_max_cyr кириллических символов), parse_price (форматы 1.013.00, 950,50). Все шаги идемпотентны (NOT EXISTS) и логируются в XLOG (START/OK/WARN/ERROR + число строк).

6a. Фронт-офис (магазин услуг) и личный кабинет

Публичная страница /UNA.md/orasldev/servouts26-shop — по образцу магазина Biro26, но продаёт услуги аутсорсинга бухгалтерии: каталог из прайс-листа модуля (CODPRICE=26, цены на сегодня), корзина, саморегистрация клиентов (физ/юр лица, IDNO 13 цифр), заказы и личный кабинет (список заказов, печатный счёт). Цены — только серверные (анти-манипуляция). Данные — собственные нормализованные таблицы SRVO_CLIENT / SRVO_ORDERS / SRVO_ORDER_ITEMS (ядро ERP TMDB_* не затрагивается — ТЗ §10.3); пароли — PBKDF2-хеши. События (регистрация, заказ, смена статуса) пишутся в XLOG.

GET  /api/servouts26/shop/catalog                      # публичный каталог
POST /api/servouts26/shop/register | login | logout    # клиент
GET  /api/servouts26/shop/me | my-orders | order/<id>
POST /api/servouts26/shop/order {items:[{cod,qty,name}], note}
GET  /api/servouts26/orders[?status=]                  # back-office
POST /api/servouts26/orders/status {order_id, status}  # NEW|CONFIRMED|IN_WORK|DONE|CANCELED

В админке — панель «Comenzi (magazin)» со сменой статусов. Сайт-обвязка WordPress: wordpress/servouts26/ (локально порт 6003, магазин в iframe на главной; см. README там).
