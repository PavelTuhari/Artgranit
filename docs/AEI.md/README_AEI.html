ОСВ — Trial Balance (Оборотно-сальдовая ведомость)

Raport contabil complet per perioadă: sold inițial + rulaj + sold final per cont. Afișare 1С-style cu subtotaluri per clasă de conturi. Invariant: Rulaj Dt = Rulaj Ct.

ColoanăSursă OracleDescriere
Sold inițial Dt/CtCTE obRulaj acumulat înainte de data_from
Rulaj perioadă Dt/CtCTE perÎnregistrări în intervalul selectat
Sold final Dt/Ctob + perCalculat prin adunare

P&L — Venituri & Cheltuieli

Filtrat automat din ОСВ: conturi clasă 8xx = VENITURI, clasă 7xx = CHELTUIELI. Afișează profit net = Venituri − Cheltuieli.

Clasă NBMTipExemple conturi
8xxxVENITURI (INCOME)8111 Dobânzi credite, 8112 Comisioane, 8113–8116
7xxxCHELTUIELI (EXPENSE)7151 Dobânzi depozite, 7211 Salarii, 7311 Chirie…
2xxxACTIVE2101 Casa, 2421 Conturi curente, 2551 Credite acordate
3xxxPASIVE / DATORII3211 Depozite, 3411 Amortizare

Schema Oracle — obiecte cu prefix AEI_

Tabele principale

  • AEI_MEMBERS — registrul membrilor
  • AEI_LOANS — contracte de credit
  • AEI_LOAN_FLOWS — grafic rambursare + plăți
  • AEI_DEPOSITS — contracte depozit
  • AEI_DEPOSIT_FLOWS — fluxuri depozit

Contabilitate

  • AEI_ACCOUNTS — planul de conturi NBM
  • AEI_JOURNAL — jurnal înregistrări duble
  • AEI_SETTINGS — parametri sistem
  • AEI_EVENT_LOG — audit trail acțiuni

Coloane cheie — AEI_LOANS

  • LOAN_ID, MEMBER_ID, CONTRACT_NO
  • PRINCIPAL, INTEREST_RATE, TERM_MONTHS
  • STATUS: ACTIVE / OVERDUE / CLOSED
  • REPAYMENT_TYPE: ANNUITY / DECLINING
  • RISK_CLASS: S / SM / SN / D / P / PI / C
  • MEMBER_TYPE: PERSON / COMPANY

Coloane cheie — AEI_LOAN_FLOWS

  • FLOW_ID, LOAN_ID, DUE_DATE
  • AMOUNT_SCHEDULED (principal + dobândă)
  • AMOUNT_PAID, PAYMENT_DATE
  • AMOUNT_OVERDUE, DAYS_OVERDUE
  • FLOW_TYPE: PAYMENT / INTEREST / FEE

Deploy — instrucțiuni

Important: deploy_to_remote.sh transferă numai codul Python/templates. DDL Oracle (modificări schemă) trebuie executat separat via python deploy_oracle_objects.py sau cu DEPLOY_ORACLE_ON_REMOTE=1.

Deploy complet (cod + restart)

# 1. Deploy cod pe server remote
bash deploy_to_remote.sh

# 2. Sau deploy manual (fișiere selectate)
rsync -avz -e "ssh -i ~/Downloads/ssh-key-2024-10-06.key" \
  app.py models/ controllers/ templates/ static/ \
  ubuntu@92.5.3.187:/home/ubuntu/artgranit/

# 3. Restart serviciu (DOAR via systemd!)
ssh -i ~/Downloads/ssh-key-2024-10-06.key ubuntu@92.5.3.187 \
  "sudo systemctl restart artgranit"

# 4. Verificare production
curl -I https://nufarul.eminescu.md/login

Verificare după deploy

  • curl -I https://nufarul.eminescu.md/login → HTTP/2 200
  • curl -s https://nufarul.eminescu.md/api/aei/dashboard → JSON cu date
  • Interfața Operator: aei-operator — dashboard cu KPI-uri
  • Interfața Backoffice: aei-backoffice — OLAP, ОСВ, P&L funcționale
  • Admin complet: aei-admin — toți membrii/credite
  • Jurnal contabil afișează intrările demo (oct 2025 – mar 2026)
  • OLAP — minim 30 credite, toate 7 dimensiuni disponibile
  • Aging — toate 4 bucket-uri: 1–30, 31–60, 61–90, >90 zile cu valori MDL
